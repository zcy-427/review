# 数据库系统的基本概念

1.**数据库DB**
长期存储在计算机内、有组织、可共享的大量数据集合

2.**数据库管理系统DBMS**
位于用户和操作系统之间，用于定义、操纵、管理和控制数据库的软件系统

3.**数据库系统DBS**
由数据库、数据库管理系统、应用系统、数据库管理员和用户等组成的系统

- **DB = 数据本身**
- **DBMS = 管理数据库的软件**
- **DBS = DB + DBMS + 应用程序 + 用户 + DBA**

# 数据模型
**数据模型**是对现实世界数据特征的抽象，用来描述数据结构、数据操作和数据约束。

数据模型尝包含3部分：

|组成|含义|
|---|---|
|数据结构|数据怎样组织|
|数据操作|可以对数据做什么操作|
|完整性约束|数据必须满足什么规则|

**常见的数据模型**：

| 模型     | 特点           |
| ------ | ------------ |
| 层次模型   | 树形结构，一对多     |
| 网状模型   | 图结构，多对多      |
| 关系模型   | 用二维表表示数据，是重点 |
| 对象关系模型 |              |
其中：层次模型与网状模型统称非关系性模型
# 关系模型的基本概念

关系模型用二维表表示数据。一个关系对应一张表，表中的行称为元组，列称为属性。
**核心概念**：

| 名词   | 含义                                         |
| ---- | ------------------------------------------ |
| 关系   | 一张二维表                                      |
| 元组   | 表中的一行                                      |
| 属性   | 表中的一列                                      |
| 域    | 属性的取值范围                                    |
| 关系模式 | 对关系结构的描述，例如 Student(Sno, Sname, Age, Dept) |
| 候选码  | 能唯一标识元组的最小属性组                              |
| 主码   | 从候选码中选定的一个                                 |
| 外码   | 一个关系中的属性引用另一个关系的主码                         |



# 关系代数入门

关系代数是一种过程化查询语言，它以关系为运算对象，通过一组关系运算得到新的关系。
