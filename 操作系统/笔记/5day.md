# 虚拟内存

虚拟内存是操作系统利用外存作为内存的扩充，使用户感觉可用内存空间比实际物理内存更大的技术。
简单说：

```text
内存不够，磁盘来凑。
```

其实现依赖两个基础：

```text
局部性原理
离散分配
```

### 局部性原理

程序运行时，在一段时间内只会访问其中一部分代码和数据。

分为：

| 类型        | 含义                  |
| --------- | ------------------- |
| **时间局部性** | 刚访问过的内容可能很快再次访问     |
| **空间局部性** | 访问某个地址后，附近地址可能很快被访问 |

例子：

- 循环语句体现时间局部性
- 数组顺序访问体现空间局部性

### 请求分页

虚拟内存常用的实现方式

```text
请求分页存储管理
```

意思：

> 程序运行时不必一次全部装入内存，只在访问某页时，如果该页不在内存，再把它调入。

### 缺页中断

如果访问的页面不在内存，就会发生缺页中断：
```text
缺页中断是指进程访问的页面不在内存中，需要由操作系统从外存调入该页而产生的中断
```

操作系统会：

```text
从外存把该页调入内存
```

如果内存满了，还要：

```text
选择一个页面淘汰
```

这就引出页面置换算法。

# 页面置换算法

题目通常给你：

```text
页面访问序列
页框数
页面置换算法
```

让你求：

```text
缺页次数
缺页率
页面置换过程
```

### 1.OPT：最佳置换算法

淘汰以后最长时间不再被访问的页面。

**例**:
访问序列：

```text
7, 0, 1, 2, 0, 3, 0, 4
```

页框数：

```text
3
```

```text
1. 缺页次数 =
2. 缺页率 =
```

### 2. FIFO：先进先出

淘汰最早进入内存的页面。

**例**：
访问序列： 

```text
1, 2, 3, 4, 1, 2, 5
```

页框数：

```text
3
```

```text
1. 每一步内存状态：
2. 缺页次数 =
3. 缺页率 =
```

### 3. LRU：最近最久未使用

淘汰最近最久没有被访问的页面。

**例**：
访问序列：

```text
1, 2, 3, 1, 4, 2, 5
```

页框数：

```text
3
```

```text
1. 缺页次数 =

```

# 文件系统

文件系统**解决的问题**是：操作系统如何组织、存储、管理和访问外存上的文件。

### 文件的逻辑结构
- 从**用户的角度**来看

| 逻辑结构  | 含义            |
| ----- | ------------- |
| 流式文件  | 文件看成一串连续字符或字节 |
| 记录式文件 | 文件由一条条记录组成    |
|       |               |

1.流式文件：普通的文本文件，二进制文件
2.记录式文件：学生成绩表，数据库记录

### 文件的物理结构

#### 1.连续分配

文件占用一组连续的磁盘块
- 优点：顺序访问和随机访问都快
- 缺点：容易产生外部碎片，不易扩展
#### 2.链接分配

文件的数据块可以分散存放，每个块中保存指向下一个块的指针。
- 优点：不产生外部碎片，易扩展
- 缺点：只能顺序访问。随机访问效率低；指针占用空间，可靠性差
#### 3.索引分配

为每个文件建立**索引表**，索引表中记录文件各数据块的地址
- 优点：支持随机访问，不产生外部碎片，文件易扩展
- 缺点：索引表需要额外的空间

### 文件控制块FCB（File Control Block）

它是操作系统管理文件的重要数据结构。

FCB 中通常包含：

```text
文件名
文件类型
文件大小
文件位置
访问权限
创建/修改时间
```

核心作用：

```text
FCB 用来描述和管理文件。
```

**文件目录实际上是由若干的PCB组成的**

### 目录

目录用于管理文件名和文件信息，操作系统通过目录找到文件。

#### 1.单级目录
所有文件都放在一个目录中。

- 优点：简单
- 缺点：不允许重名，查找效率低，不适合多用户。
#### 2.两级目录
分为：

```text
主文件目录 MFD
用户文件目录 UFD
```

每个用户有自己的目录。

- 优点：不同的用户可以拥有同名文件
- 缺点：目录层次依旧有限
#### 3.树形目录
类似 Windows 文件夹结构：

```text
C:/用户/文档/文件.txt
```

优点：层次清晰，便于分类管理，允许不同目录下文件重名


### 路径名

| 路径   | 含义         |
| ---- | ---------- |
| 绝对路径 | 从根目录开始的路径  |
| 相对路径 | 从当前目录开始的路径 |
