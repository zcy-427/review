**1.什么是操作系统？**
操作系统是配置在计算机硬件上的第一层系统软件，它用于管理和控制计算机的硬件与软件资源，合理组织计算机的工作流程，并为用户和应用程序提供方便的接口。

**2.操作系统解决的问题**
如何**高效、安全、方便**地管理计算机资源的问题。

**3.操作系统的主要功能**

| 功能    | 说明                  |
| ----- | ------------------- |
| 处理机管理 | 管理 CPU，决定哪个进程运行     |
| 存储器管理 | 管理内存分配、回收、保护        |
| 文件管理  | 管理文件的创建、删除、读写       |
| 设备管理  | 管理输入输出设备            |
| 用户接口  | 给用户和应用程序提供使用 OS 的方式 |

4.**操作系统的4大特征**

| 特征  | 关键词                                             | 例子            |
| --- | ----------------------------------------------- | ------------- |
| 并发  | 两个或多个事件在 **同一时间间隔内** 发生                         | 一边听歌一边下载      |
| 共享  | 系统中的资源可供多个并发执行的进程共同使用                           | 多个程序共用打印机     |
| 虚拟  | 虚拟性是指通过某种技术，把一个物理实体变成多个逻辑上的对应物                  | 虚拟内存          |
| 异步  | 在多道程序环境下，程序的执行以不可预知的速度推进，即程序会因为资源、I/O、调度等原因走走停停 | 进程可能因为 I/O 暂停 |

**5.并发 vs 并行**

|项目|并发|并行|
|---|---|---|
|关键词|同一时间段|同一时刻|
|执行方式|交替执行|同时执行|
|单核 CPU|可以|不可以|
|多核 CPU|可以|可以|

**6.进程三态转换**
三种基本状态

```text
就绪态：等待 CPU
运行态：正在用 CPU
阻塞态：等待事件 / I/O
```

状态转换图

```text
就绪态 --被调度--> 运行态
运行态 --时间片到--> 就绪态
运行态 --等待 I/O--> 阻塞态
阻塞态 --I/O 完成--> 就绪态
```

**7.程序和进程的区别**
**程序**

> 程序是完成某一功能的一组指令的有序集合，是静态的。

**进程**

> 进程是程序的一次执行过程，是系统进行资源分配和调度的基本单位。

**一句话**：程序是静态的指令集合，进程是程序的一次执行过程。进程是动态的，具有 PCB 和运行状态，并占有一定系统资源；程序本身没有运行状态。

**8.进程状态转换**

进程状态转换解决的是：**操作系统如何描述一个进程当前处于什么执行阶段。**
```text
等 CPU：就绪态
用 CPU：运行态
等事件 / 等 I/O：阻塞态
```
判断进程的状态
```text
第一步：它正在用 CPU 吗？
是 → 运行态

第二步：它没用 CPU，但是否已经具备运行条件，只差 CPU？
是 → 就绪态

第三步：它是否在等待 I/O 或某个事件？
是 → 阻塞态
```

**9.CPU调度算法-FCFS（先来先服务）**

按进程到达的先后顺序进行调度，谁先到，谁先执行

**四个概念:**

到达时间 = 进程来到系统的时间
服务时间 = 进程需要 CPU 运行多久
完成时间 = 进程执行结束的时间
周转时间 = 完成时间 - 到达时间
带权周转时间 = 周转时间 ÷ 服务时间
平均周转时间 = 周转时间 ÷ 进程数
平均带权周转时间 = 带权周转时间 ÷ 进程数

**例题：**
有 4 个进程：

|进程|到达时间|服务时间|
|---|---|---|
|P1|0|3|
|P2|2|6|
|P3|4|4|
|P4|6|2|

采用 **FCFS 先来先服务算法**。

4. 每个进程的周转时间是多少？

**10.CPU调度算法-SJF（短优先作业）**

为了减少平均等待时间，优先执行服务时间短的进程

SJF 有两种：

|类型|含义|
|---|---|
|非抢占式 SJF|一旦开始执行，就一直运行到完成|
|抢占式 SJF / SRTF|如果来了更短的进程，可能抢占当前进程|

**12.CPU调度算法-时间片轮转 RR**

时间片轮转调度算法将 CPU 时间划分为固定长度的时间片，按照就绪队列顺序依次分配给进程。若进程在一个时间片内未完成，则被剥夺 CPU 并排到就绪队列队尾；若提前完成，则立即调度下一个进程。
