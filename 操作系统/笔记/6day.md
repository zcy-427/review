# I/O管理

**解决的问题**：操作系统如何控制和管理各种输入输出设备，提高设备利用率，并为用户提供统一方便的接口。

### I/O的控制方式

常见四种：

```text
程序直接控制方式
中断驱动方式
DMA 方式
通道方式
```

#### 1.程序直接控制方式

CPU不断查询设备状态，也叫做**轮询方式**

特点：CPU一直等设备，效率低
#### 2.中断驱动方式

CPU 启动 I/O 后，可以去做别的事。  设备完成后，通过中断通知 CPU。

特点：比轮询的效率高
#### 3.DMA方式（Direct Memory Access）

又叫作：直接存储器访问

数据在设备与内存之间直接传送，无需CPU逐字节干预。CPU只在开始和结束时参与。

#### 4.通道方式

通道是一种专门负责 I/O 的处理机。

特点：进一步减轻CPU的负担。一般比DMA更高级

### 缓冲技术

```text
缓和 CPU 和 I/O 设备速度不匹配的问题，提高并行性。
```

常见：

```text
单缓冲
双缓冲
循环缓冲
缓冲池
```
