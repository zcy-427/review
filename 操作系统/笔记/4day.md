# 内存管理

> 解决操作系统如何为程序分配、回收、保护和扩充内存空间。


### 1.内存管理的主要功能

| 功能   | 含义             |
| ---- | -------------- |
| 内存分配 | 给进程分配内存空间      |
| 内存回收 | 进程结束后收回内存      |
| 地址转换 | 把逻辑地址转换成物理地址   |
| 内存保护 | 防止进程越界访问       |
| 内存扩充 | 利用虚拟存储技术扩大逻辑内存 |

---

### 2.几个基础概念

**逻辑地址**

> 程序中使用的地址，也叫相对地址或虚地址。

简单说：

```text
程序自己看到的地址。
```

**物理地址**

> 内存中实际的地址，也叫绝对地址。

简单说：

```text
真实内存里的地址。
```

**地址重定位**

> 把逻辑地址转换为物理地址的过程。

公式大致理解：

```text
物理地址 = 起始地址 + 逻辑地址
```

---

### 3.连续分配方式

常见方式：

|方式|含义|
|---|---|
|单一连续分配|内存中只装一个用户程序|
|固定分区分配|内存预先划成固定大小分区|
|动态分区分配|按进程需要动态划分内存|

---
**固定分区分配**

提前把内存分成几个固定大小区域。

缺点：

```text
容易产生内部碎片
```

**内部碎片**

> 已分配给进程但没有被进程使用的空间。

例子：

```text
分区大小 100KB
进程只需要 80KB
剩下 20KB 不能给别人用
```

这 20KB 就是内部碎片。

---

**动态分区分配**

按进程实际需要分配内存。

缺点：

```text
容易产生外部碎片
```

**外部碎片**

> 分布在已分配分区之间、无法被充分利用的小空闲区。

例子：

```text
空闲区有 10KB、20KB、15KB
总共 45KB
但进程需要连续 40KB
如果没有连续 40KB，就无法分配
```

这就是外部碎片问题。

---
### 4.内部碎片 vs 外部碎片

这个非常常考。

| 对比    | 内部碎片        | 外部碎片     |
| ----- | ----------- | -------- |
| 位置    | 已分配分区内部     | 分区之间的空闲区 |
| 原因    | 分配给进程的空间用不完 | 空闲空间不连续  |
| 常见于   | 固定分区、分页     | 动态分区、分段  |
| 是否已分配 | 已分配         | 未分配但难利用  |

口诀：

```text
内部：分给你了但你没用完。
外部：还没分出去但太零散。
```

# 动态分区分配算法

### 1. 首次适应算法 First Fit

从内存低地址开始查找，找到第一个能满足要求的空闲分区就分配。
- 优点：查找速度块
- 缺点：低地址部分容易产生外部碎片

### 2. 最佳适应算法 Best Fit

从所有能满足要求的空闲分区中，选择最小的那一个。
- 优点：尽量减少浪费
- 缺点：容易产生很多很小的外部碎片，是3个算法当中产生最多的

### 3. 最坏适应算法 Worst Fit

从所有能满足要求的空闲分区中，选择最大的那一个。
- 优点：剩余空间较大，可能还能继续使用
- 缺点：大空闲区容易被快速划分

# 分页和分段

两者都不是连续分配，而是可以让程序分散到内存当中

## 1.分页

#### **概念**

把进程的逻辑地址空间分成大小相等的页，把内存物理空间分成大小相等的页框，然后按页为单位进行分配。

简单来说：
```text
程序切成等大的页；
内存切成等大的块；
页装进页框。
```
分页地址通常分成两部分：

```text
页号 + 页内偏移量
```

|部分|含义|
|---|---|
|页号|逻辑地址属于第几页|
|页内偏移量|在该页内的位置|

通过**页表**，把页号转换成页框号。

- 页表：记录页号和物理页框号对应关系的数据结构。
比如：

|页号|页框号|
|---|---|
|0|5|
|1|2|
|2|8|

意思：

```text
逻辑第 0 页放在物理第 5 个页框
逻辑第 1 页放在物理第 2 个页框
逻辑第 2 页放在物理第 8 个页框
```

#### **分页地址转换**

公式：

```text
逻辑地址 = 页号 + 页内偏移量
页表查页号 → 页框号
物理地址 = 页框号 × 页大小 + 页内偏移量
```

如果题目直接给页号和偏移量，用这个公式即可。

**例题**

页大小为：

```text
1KB = 1024B
```

逻辑地址中：

```text
页号 = 2
页内偏移 = 100
```

页表显示：

```text
页 2 → 页框 5
```

求物理地址。

公式：

```text
物理地址 = 页框号 × 页大小 + 页内偏移
```

代入：

```text
物理地址 = 5 × 1024 + 100 = 5220
```

## 2.分页的特点

#### 概念

> 按程序的逻辑结构，把程序划分为若干个大小不等的段，每段可以独立装入内存。

简单说：

```text
按功能模块分。
```
一个程序可以分成：

```text
代码段
数据段
堆栈段
主程序段
子程序段
```

这些段大小通常不一样。

#### 分段地址结构

分段地址通常是：

```text
段号 + 段内偏移量
```

段表记录：

```text
段号 → 段基址 + 段长
```

|项目|含义|
|---|---|
|段基址|该段在内存中的起始地址|
|段长|该段的长度|
|段内偏移|在该段内部的位置|

地址转换：

```text
如果 段内偏移 < 段长：
    物理地址 = 段基址 + 段内偏移
否则：
    越界错误
```
