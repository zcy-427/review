# 内存管理


### 1.内存管理的主要功能

| 功能   | 含义             |
| ---- | -------------- |
| 内存分配 | 给进程分配内存空间      |
| 内存回收 | 进程结束后收回内存      |
| 地址转换 | 把逻辑地址转换成物理地址   |
| 内存保护 | 防止进程越界访问       |
| 内存扩充 | 利用虚拟存储技术扩大逻辑内存 |

---

### 2.几个基础概念

**逻辑地址**

> 程序中使用的地址，也叫相对地址或虚地址。

简单说：

```text
程序自己看到的地址。
```

**物理地址**

> 内存中实际的地址，也叫绝对地址。

简单说：

```text
真实内存里的地址。
```

**地址重定位**

> 把逻辑地址转换为物理地址的过程。

公式大致理解：

```text
物理地址 = 起始地址 + 逻辑地址
```

---

### 3.连续分配方式

常见方式：

|方式|含义|
|---|---|
|单一连续分配|内存中只装一个用户程序|
|固定分区分配|内存预先划成固定大小分区|
|动态分区分配|按进程需要动态划分内存|

---
**固定分区分配**

提前把内存分成几个固定大小区域。

缺点：

```text
容易产生内部碎片
```

**内部碎片**

> 已分配给进程但没有被进程使用的空间。

例子：

```text
分区大小 100KB
进程只需要 80KB
剩下 20KB 不能给别人用
```

这 20KB 就是内部碎片。

---

**动态分区分配**

按进程实际需要分配内存。

缺点：

```text
容易产生外部碎片
```

**外部碎片**

> 分布在已分配分区之间、无法被充分利用的小空闲区。

例子：

```text
空闲区有 10KB、20KB、15KB
总共 45KB
但进程需要连续 40KB
如果没有连续 40KB，就无法分配
```

这就是外部碎片问题。
